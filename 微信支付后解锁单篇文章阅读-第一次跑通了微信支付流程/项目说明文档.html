📖 项目简介

这是一个 WordPress 插件，实现微信支付扫码付费后解锁单篇文章的功能。 用户在 WordPress 文章中看到付费内容时，通过扫描微信支付二维码完成付款（固定 0.10 元）， 支付成功后页面自动刷新并展示完整文章内容。无需依赖 WooCommerce、MemberPress 等复杂会员插件， 是一个轻量级的文章付费阅读解决方案。

🔄 端到端业务流程

  1. 作者发布文章
    在 WordPress 文章中使用 [pay_content]...[/pay_content] Shortcode 包裹付费内容。
  2. 用户访问文章
    Shortcode 回调检查用户是否已有支付 Cookie。若有，直接展示内容；若无，进入支付流程。
  3. 创建微信支付订单
    插件调用微信统一下单 API /pay/unifiedorder，发送签名后的 XML 请求（金额 0.10 元、商品描述为文章标题、唯一订单号 WX+时间戳+随机数）。
  4. 展示二维码
    微信返回 code_url，前端使用 QRCode.js 生成二维码。桌面端直接扫码，移动端提示复制链接。
  5. 用户扫码支付
    用户打开微信扫描二维码，在微信内完成支付。
  6. 微信异步通知
    微信服务器向 /wechat-pay-notify 发送 POST 请求。插件验证 MD5 签名后，将订单状态从 pending 更新为 paid，缓存回调数据，并调度 WP Cron 异步任务。
  7. 前端轮询检测到支付成功
    浏览器每 2 秒 AJAX 查询订单状态。一旦检测到 paid，设置 7 天有效 Cookie，显示支付成功提示并刷新页面。
  8. 页面刷新 — 内容解锁
    刷新后 Cookie 存在，Shortcode 直接渲染完整付费内容。
  9. 异步发送邮件
    WP Cron 在回调 1 秒后触发，读取缓存的支付数据，发送包含订单号、金额、支付时间、购买者 OpenID、文章链接的 HTML 邮件给管理员。

idINT (自增主键)记录 ID

post_idBIGINT关联的文章 ID

statusVARCHAR(20)订单状态：pending / paid
created_atDATETIME创建时间

⚙ 配置说明

在 wechat-pay-simple.php 文件头部修改以下常量：

define('WECHAT_PAY_APPID',     'your_appid');       // 微信商户 AppID
define('WECHAT_PAY_MCHID',     'your_mchid');       // 微信商户号
define('WECHAT_PAY_API_KEY',   'your_api_key');     // 微信商户 API 密钥
define('WECHAT_PAY_TOTAL_FEE', '0.10');             // 固定支付金额（元）

安装方式：将 wechat-pay-simple.php 和 wechat-pay-email.php 放入 wp-content/plugins/wechat-pay-simple/ 目录，在 WordPress 后台启用插件即可。

📝 使用方式

在文章编辑器中，用 Shortcode 包裹需要付费的内容：

[pay_content]
这里是需要付费才能阅读的精彩内容...
可以包含文字、图片、任何 HTML。
[/pay_content]

免费内容直接写在 Shortcode 外面即可。
