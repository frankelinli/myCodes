

好问题 👌，一篇文章的 **description**（描述）对 SEO、社交分享和读者点击率都很重要，它相当于是文章的“摘要”或“广告语”。

📖 什么是 description？

在网页中，通常会输出到 `<meta name="description">` 标签，比如：

html



```xml
<meta name="description" content="介绍如何用 remark 和 unified 生态在 Node.js 中处理 Markdown，包括 frontmatter、slug、目录等增强功能。">
```

==搜索引擎会把这段内容作为 **搜索结果摘要** 展示（除非它认为你的正文里有更相关的内容）。==





------

## ✍️ 写 description 的原则

1. **简洁明了**
    推荐长度：
   - **150–160 字符**（英文/拉丁字符计算）
   - 中文也差不多两三句话（40–50 汉字左右）。超过会被截断。
2. **包含关键词**
    避免堆砌，但要自然地包含文章的核心主题关键词。
    例如：文章是“remark 处理 Markdown”，最好在 description 中出现这几个词。
3. **吸引点击**
   - 像广告文案一样，告诉读者“这篇文章能解决你什么问题”。
   - 使用主动语气，例如“本篇文章将带你快速上手…”
4. **独立唯一**
    站点中每篇文章的 description 应该不一样，避免重复。

------

## 📋 示例

### 技术教程类文章

yaml



```yaml
description: 本文介绍如何使用 remark 和 unified 生态在 Node.js 中处理 Markdown，包括 frontmatter 解析、slug 生成、目录自动化等功能。
```

### 经验分享/软文

yaml



```yaml
description: 写 Markdown 文章时总感觉功能不够用？这篇文章带你用 remark 插件解锁目录、公式、代码高亮等多种增强能力。
```

### 新闻/博客更新

yaml



```yaml
description: 2025 年 9 月，Node.js 在 Markdown 生态上继续发力，本文分享了 remark/unified 插件的应用场景与实践经验。
```

------

## ✅ 总结

一篇文章的 **description** 应该做到：

- **简短（150–160 字符以内）**
- **包含核心关键词**
- **清楚表达文章价值/亮点**
- **每篇文章唯一**



## 📖 真实案例对比

### 1. 技术博客

**文章主题**：如何用 Docker 部署 WordPress
 **常见错误写法**（自动截取正文前 150 字）：

> 本文记录了我最近在服务器上折腾 Docker 部署的过程，遇到了一些小坑，比如权限问题、数据库初始化失败，后来我尝试了几种方法……

缺点 → 太随意，没有关键词，不利于 SEO。

**优化后的 description**（手写的 meta 描述）：

> 本文手把手演示如何使用 Docker 部署 WordPress，包括容器配置、数据库连接、数据卷管理以及常见问题的解决方法，帮助你快速搭建生产可用的网站环境。

👉 特点：

- 清楚点出关键词（**Docker 部署 WordPress**）
- 概括文章价值（快速搭建生产可用环境）
- 150 字左右

------

### 2. 新闻/信息文章

**文章主题**：苹果发布 iPhone 16
 **错误写法**（截取导语）：

> 北京时间 9 月 15 日凌晨，苹果公司在秋季发布会上推出了全新的 iPhone 16、Apple Watch Series 11……

**优化写法**：

> 苹果在 2025 年秋季发布会推出 iPhone 16，搭载 A19 芯片、全新相机系统和更长续航。本篇整理了 iPhone 16 的配置亮点、售价和首发上市信息。

👉 特点：

- 概要总结，快速传达信息
- 含关键词（**iPhone 16、配置、售价**）
- 吸引人点进去看细节

------

### 3. 个人经验分享

**文章主题**：自由职业接单经验
 **错误写法**（正文截取）：

> 我从 2023 年开始尝试自由职业，接单主要集中在前端开发和写作，最开始的时候收入并不多，但慢慢地……

**优化写法**：

> 分享我两年自由职业的接单经验：如何寻找客户、谈判价钱、避免踩坑，以及让收入稳定增长的实用方法，适合想要探索远程工作和独立职业的朋友。

👉 特点：

- 明确关键词（**自由职业、接单经验**）
- 内容亮点列表化（寻找客户、谈价、避坑）
- 对目标人群有吸引力

------

## ✅ 总结写法公式

写 `description` 时可以套这个公式：

**[核心主题关键词] + [文章内容范围] + [价值/亮点]**

举例：

- “本文介绍 **remark 插件在 Markdown 中的用法**，包括 frontmatter 解析、标题锚点、目录自动生成和代码高亮，帮助你快速提升写作体验。”
- “一份完整的 **自由职业指南**：从客户获取、项目管理到财务规划，适合想进入远程工作的人。”
